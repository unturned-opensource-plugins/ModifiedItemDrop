# ModifiedItemDrop v2.0.0 Release Notes

## Breaking upgrade notice

ModifiedItemDrop v2.0.0 is an intentional breaking reliability redesign. Operators must review configuration and command usage before deployment.

- v1 `RuleSet` death-processing configuration is not auto-migrated. Rewrite rules using v2 `OutcomeRulesXml` and the examples in `docs/migration/v1-to-v2-configuration.md`.
- v1 flat `/mid` commands are removed. Use the grouped v2 command surface documented in README and the migration guide.
- v2 Durable Claim storage uses `claims/v2/claims.json`; v1 root `claims.json` is not imported, deleted, or rewritten.

## Highlights

- Canonical Player Asset Outcome model for Drop, Keep, Delete, and Respawn Grant decisions.
- Declarative nested XML Outcome Rules with priority, explicit targets, probabilities, and catch-all validation.
- Durable Claim persistence with backup/corrupt-path recoverability diagnostics.
- Safe mode for invalid Outcome Rules and degraded mode for untrusted Claim storage.
- Grouped v2 command surface:
  - `/mid config reload`
  - `/mid rules preview [player]`
  - `/mid rules explain slot <PlayerAssetSlot>`
  - `/mid rules explain item <itemId>`
  - `/mid inventory dump [player]`
  - `/mid claims list [player]`
  - `/mid claims recover oldest|all`
  - `/mid diagnostics status`
  - `/mid diagnostics export`
